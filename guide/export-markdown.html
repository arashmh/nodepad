Export Markdown

Action

Exports the current project as a richly formatted .md file. The output is designed to render well in GitHub, Obsidian, Notion, and LLM context windows.

How to Trigger

  • Press Ctrl+K → select Export markdown

The browser downloads a .md file (filename = slugified project name).

Document Structure

  1. YAML front matter — title, date, nodes count, types list, source: nodepad
  2. Title block — # Project Name with a summary line (date + node count)
  3. Overview table — a markdown table listing each content type with its emoji and count
  4. Pinned nodes callout — if any notes are pinned, they appear in a blockquote list at the top
  5. Table of contents — linked list of type sections with counts
  6. Per-type sections (research-logical order)
  7. Footer — nodepad attribution and date

Type-Specific Formatting

Claims → rendered as a markdown table with columns: Claim, Category, Confidence (visual bar like ███░░ 60%), AI Annotation.

Tasks → rendered as GFM task lists (- [ ] **task text**) with category and annotation as sub-items.

Quotes → rendered as blockquotes with attribution and annotation in italics.

All other types → rendered as ### headings with metadata, AI annotation as a nested blockquote, and sources as linked list.

Section Order

Types appear in research-logical order:

Thesis → Claims → Questions → Ideas → Tasks → Entities → Definitions → References → Quotes → Opinions → Reflections → Narratives → Comparisons → Notes
