Export .nodepad

Action

Exports the current project as a .nodepad file — a versioned JSON snapshot that captures everything: notes, annotations, connections, ghost notes, and all AI enrichment data.

How to Trigger

  • Press Ctrl+K → select Export .nodepad

The browser downloads the file immediately.

File Format

A .nodepad file is pretty-printed JSON (2-space indent) with the .nodepad extension. It's standard JSON, just with a custom extension.

What's Included

FieldDescription
versionSchema version (currently 1) for future migrations
exportedAtUnix timestamp of the export
project.idThe project's unique ID
project.nameProject name (used for the filename)
project.blocks[]Every note: text, type, category, annotation, confidence, sources, connections, pin state, sub-tasks
project.collapsedIdsWhich notes are collapsed in the UI
project.ghostNotesAI ghost notes (id, text, category)
project.lastGhostTextsRecent ghost texts (for de-duplication on re-import)

What's Stripped

Transient UI state (isEnriching, isError, statusText) is omitted — only persistent data is saved. Ghost notes always have isGenerating: false.

Filename

The project name is slugified (lowercased, spaces→hyphens, non-alphanumeric stripped, max 60 chars), e.g. my-research.nodepad. Falls back to project.nodepad.
