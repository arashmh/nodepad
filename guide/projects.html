Projects

Navigate

The Projects command opens the Project Sidebar on the left side of the screen. Internally, projects are called "Spaces" — this is the label shown in the sidebar header.

How to Open

  • Press Ctrl+K → select Projects
  • Click the menu icon (hamburger) in the top-left status bar

What You See

Each project entry shows:

  • The project name in monospace bold (highlighted in accent colour if active)
  • A node count below the name (e.g. "12 nodes")

Switching Projects

Click any project in the list to make it active. The canvas immediately updates to show that project's notes, ghost notes, and collapsed state. Any pending AI enrichment from the previous project is safely cleared.

Renaming

Hover over a project and click the pencil icon. The name becomes an inline text input. Press Enter or click away to confirm, Escape to cancel.

Deleting

Hover and click the trash icon (only available if more than one project exists). A red confirmation overlay appears inline: "Delete Space?" with confirm/cancel buttons. If you delete the active project, the app switches to the first remaining project.

Sidebar Footer

At the bottom of the sidebar you'll find:

  • New Space — create a new project
  • Import .nodepad — open a file picker to import a saved project
  • Settings — slide to the AI settings panel (API key, model, web grounding)

Data Persistence

All projects are saved to localStorage under nodepad-projects, with a rolling backup under nodepad-backup. On load, the app tries the primary key first, falls back to the backup, then attempts migration from a legacy single-project format, and finally falls back to the default starter project.
