Tiling View

View

The Tiling view arranges your notes as a space-filling mosaic using a Binary Space Partitioning (BSP) tree. Think of it like a window tiling manager — the available area is recursively halved, alternating between vertical and horizontal splits, until every note fills exactly one rectangle. No whitespace is wasted.

Layout Algorithm

Notes are laid out using a BSP approach: the viewport is split in half (vertically), then each half is split horizontally, and so on. Every note gets a tile that stretches to fill its allocated rectangle. The result is a dense mosaic where all visible space is used.

Pagination

Notes are grouped into pages of 7 tiles each. Pages stack vertically and you scroll through them.

Sorting & Pinning

Notes are sorted with pinned notes first, then by age (oldest first). Pinning a note promotes it to the front of the list, giving it the most prominent position on page 1.

Task Sticky Header

If any task-type note exists, it's pulled out of the mosaic and displayed as a sticky header pinned to the top of the view. It remains visible as you scroll. This is the only card in tiling view that supports collapse/expand.

Connection Highlighting

Hover a note's connection indicator → unrelated tiles dim to 15% opacity.
Click the indicator → the highlight locks in place.
Escape or click the background → clears the lock.

Tile Interactions

  • Edit note text inline
  • Edit the AI annotation
  • Delete the note
  • Re-enrich (re-run AI analysis)
  • Change content type manually
  • Toggle pin
  • Toggle / delete sub-tasks (for task-type notes)

There is no drag-and-drop and no manual resize — positions and sizes are fully automatic.

Minimap

A floating minimap appears in the bottom-right corner when content overflows. Each page is shown as a button with colored dots (matching note content types). Hover for a tooltip listing that page's notes; click to smooth-scroll to that page.
