Import .nodepad

Action

Imports a previously exported .nodepad file as a new project. It never overwrites existing projects.

How to Trigger

  • Press Ctrl+K → select Import .nodepad
  • Open the sidebar → click Import .nodepad in the footer

A file picker opens, accepting .nodepad and .json files.

What Happens on Import

  1. The file is read and parsed as JSON.
  2. Three validation checks run:
    • Content must be valid JSON
    • Root must be a non-null object
    • Must contain project.blocks as an array
  3. If any check fails, an error message is shown via alert.

Always Creates a New Project

A fresh ID is generated so there are no collisions with existing projects. If the imported project name already exists, it gets a copy suffix: "My Research (2)", "My Research (3)", etc.

The new project is appended to the project list and immediately becomes active.

Post-Import Cleanup

Transient UI flags on every note are reset — isEnriching: false, isError: false, statusText: undefined — so no stale enrichment state carries over from the export.

The file input is also reset after import, so the same file can be re-imported again if desired.
