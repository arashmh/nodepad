Graph View

View

The Graph view renders your notes as a force-directed network diagram powered by D3.js. Notes become nodes, and AI-discovered connections become curved arcs between them. The most connected notes gravitate toward the centre; isolated notes drift to the edges.

How Nodes Are Positioned

A physics simulation places nodes using several forces working together:

  • Radial centrality — nodes with more connections are pulled toward the centre; isolated nodes are pushed to the outer ring.
  • Link attraction — connected nodes attract each other (~120px apart).
  • Charge repulsion — all nodes repel each other to prevent overlap.
  • Collision — nodes can't physically overlap (padding enforced).
  • Gravity — a weak centering force keeps the graph from drifting off-screen.

Connection Lines

  • Curved arcs — regular influencedBy relationships. The curve bows outward from the centre so overlapping links are distinguishable.
  • Dashed faint lines — synthesis links connecting the AI ghost node to every note (very subtle at 5% opacity).

Hover Behaviour

Hover a node → the node and its direct neighbours stay opaque; everything else fades to ~8% opacity (spotlight effect).

A floating tooltip appears near your cursor showing the content type, category, link count, and full note text.

Click & Detail Panel

Click a node to select it. The graph shrinks to 70% width and a detail panel slides in from the right (30% width). The panel shows:

  • Colour-coded header with type icon, type label, category, and date
  • Action buttons: reclassify type, pin, re-enrich, close
  • Full note text (double-click to edit inline)
  • Confidence bar showing AI classification confidence
  • AI annotation (rendered as Markdown, double-click to edit)
  • Sources / URLs if web grounding is active
  • Connected nodes list — click any to navigate to that node
