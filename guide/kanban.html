Kanban View

View

The Kanban view organizes all your notes into columns grouped by content type. Each note's AI-assigned type (task, claim, question, idea, entity, quote, etc.) determines which column it appears in.

Column Grouping

There are 14 possible content types. Only columns that contain at least one note are shown. A special "Enriching" column appears for notes currently being processed by the AI.

Column Order

Columns follow a fixed priority:

  1. Tasks (always first)
  2. Thesis
  3. Enriching (if any)
  4. All remaining types alphabetically

Each column header shows an icon, the type name in uppercase, and a card count.

No Drag-and-Drop

Columns are read-only groupings. To move a note to a different column, change its content type via the card's type picker — the note automatically re-sorts into the appropriate column.

Scrolling

  • Horizontal — the entire board scrolls left/right to reveal columns beyond the viewport.
  • Vertical — each column scrolls independently, so a tall column doesn't push other columns down.

Connection Highlighting

Hover over a card's connection indicator to dim all unrelated cards across every column. Click to lock the highlight so it persists while you explore. Click again to unlock.

Minimap

A floating minimap in the bottom-right corner shows one icon per visible column. Hover for the column name and card count; click to smooth-scroll the board so that column is centered. Useful when many columns extend beyond the viewport.
